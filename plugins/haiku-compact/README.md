# haiku-compact

Make Claude Code's **context compaction** run on a cheap Haiku model without
changing the model you use for normal turns.

Compaction summarizes the conversation when the context window fills. By default
that summary is generated by your main model (Opus/Sonnet), which is expensive.
This plugin reroutes **only** the compaction request to Haiku.

## Why a proxy, and not a hook

A `PreCompact` hook *cannot* change the compaction model. Claude Code reads the
compaction model from in-memory app state (`mainLoopModel`), not from
`settings.json`, and re-reads it on every turn; no hook output can override it.
Editing `settings.json` mid-session is a no-op for the running session (and can
strand you on Haiku if the restore step never runs). The only reliable place to
swap the model is **on the wire**, between Claude Code and the API.

So `haiku-compact` is a tiny localhost reverse proxy. It forwards every request
untouched **except** the compaction summarization call -- which it recognizes by
the fixed instruction in the request's final message -- and rewrites that one
request's `model` field to Haiku. Responses stream straight through.

## Setup

Two ways to run it. Both need the proxy running and `ANTHROPIC_BASE_URL` pointed
at it.

### Option A: launcher (zero config)

Run Claude Code through the bundled launcher. It starts an ephemeral proxy, points
`ANTHROPIC_BASE_URL` at it, and execs `claude` (forwarding all arguments):

```bash
"${CLAUDE_PLUGIN_ROOT}/build/haiku-compact" launch -- claude
```

(Set a shell alias, e.g. `alias cc='haiku-compact launch -- claude'`.)

### Option B: always-on daemon + settings env

The plugin's `SessionStart` hook already starts the proxy daemon automatically
(idempotent; it self-exits after an idle period). You just have to point Claude
Code at it once, by adding this to your `settings.json`:

```json
{
  "env": {
    "ANTHROPIC_BASE_URL": "http://127.0.0.1:8788"
  }
}
```

That's it. The daemon listens on `127.0.0.1:8788` by default.

## Verifying it works

The proxy logs each swap to `${TMPDIR}/haiku-compact.log`:

```
haiku-compact 2026/06/13 21:03:00 compaction detected: swapped model to claude-haiku-4-5-20251001 (148231 bytes)
```

## Configuration

All optional, via environment variables:

| Variable | Default | Purpose |
|----------|---------|---------|
| `HAIKU_COMPACT_PORT` | `8788` | Daemon listen port |
| `HAIKU_COMPACT_UPSTREAM` | `https://api.anthropic.com` | Real API endpoint (set this if you use a gateway) |
| `HAIKU_COMPACT_MODEL` | `claude-haiku-4-5-20251001` | Model to swap in |
| `HAIKU_COMPACT_MAX_INPUT_BYTES` | `700000` | Skip the swap if the request body is larger than this (see Caveats); `0` disables the guard |
| `HAIKU_COMPACT_IDLE_MINUTES` | `60` | Daemon self-exits after this much inactivity; `0` keeps it running |
| `HAIKU_COMPACT_LOG` | `${TMPDIR}/haiku-compact.log` | Log file path |

## Commands

```bash
haiku-compact serve     # run the proxy in the foreground
haiku-compact daemon    # start it detached if not already running (used by the hook)
haiku-compact launch -- claude [args...]
haiku-compact stop      # stop the daemon
```

## Caveats

- **Context window.** Haiku 4.5 has a 200k-token context window. If you run a
  large-context model (e.g. a 1M-token session) and compaction fires far above
  ~200k tokens, Haiku could not accept it. The `HAIKU_COMPACT_MAX_INPUT_BYTES`
  guard skips the swap (falling back to your main model) when the request is too
  large, so compaction never breaks. Standard 200k sessions never hit this.
- **No clamping needed.** Claude Code caps the compaction summary at 20k output
  tokens, well under Haiku 4.5's limit, so only the `model` field is rewritten.
- **Auth is untouched.** Your API key / OAuth headers are forwarded verbatim over
  TLS to the real endpoint. The proxy binds `127.0.0.1` only and never logs
  request bodies or headers.
- **Fail-open.** Any parse/forward error leaves the request unmodified -- a broken
  proxy degrades to "no savings", never to "broken compaction".
